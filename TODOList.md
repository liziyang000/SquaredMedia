# 网站小功能 TODO

> 记录日期：2026-07-23
>
> 范围：以 React 前台和现有 `pingfangapi` 能力为基础，优先收录可独立完成的小迭代。
>
> 不包含：整体改版、SEO 切流、会员/支付体系、播放器内核替换或绕过 MacCMS 登录与播放权限的方案。

当前已有收藏、观看记录、继续观看、评分、留言/报错等基础能力；以下项目以补足使用体验为主，避免重复建设。

## 可优先完成（无需新增服务端数据）

- [ ] 搜索历史与快捷回搜
  - 在 `/search` 保存最近 10 个有效搜索词，并提供一键清空。
  - 历史仅保存在当前浏览器，不上传用户搜索记录。

- [ ] 搜索无结果时的回退入口
  - 给出“清除筛选条件”“浏览全部分类”“查看年度热榜”三个明确入口，避免空白页。

- [ ] 筛选条件摘要与一键重置
  - 在影片库和分类页显示已选地区、年份、类型等条件；支持一次清除非必要筛选，保留当前页面路径。

- [ ] 影片详情页复制链接
  - 复制干净的详情页地址（`/vod/{id}`），显示短暂成功提示；不复制播放 iframe 或任何权限参数。

- [ ] 本地观看记录单条删除
  - 在 `/history` 为每条记录提供删除操作，不再只能一次清空全部。
  - 只影响当前浏览器的本地记录，不影响登录账号的播放记录。

- [ ] 影片库卡片密度切换
  - 提供“标准 / 紧凑”两种卡片视图，并将偏好保存在本地。
  - 移动端保持单列或现有响应式布局，不能产生横向滚动。

- [ ] 剧情简介展开与收起
  - 长简介默认展示有限行数，用户可手动展开；短简介不显示多余按钮。

## 可复用现有账号与内容接口

- [ ] 首页“继续观看”进度条与单条隐藏
  - 用已有观看位置和时长展示直观进度；支持从首页移除某一条记录。
  - 登录用户复用播放记录删除能力，游客只删除本地记录。

- [ ] 收藏页站内筛选
  - 按影片名、备注或收藏时间在已加载收藏中快速筛选，不新增服务端搜索接口。
  - 保持现有全选、批量删除和单条删除行为。

- [ ] 反馈与片源报错的快捷分类
  - 在现有表单增加“播放问题、字幕/翻译、资料错误、页面体验、其他”等选项，并把分类写入留言正文前缀。
  - 不改变现有审核、验证码和登录要求。

- [ ] 详情页关联内容的“换一批”
  - 在已有关联影片中随机/轮换展示一组内容，减少用户反复进入详情页时看到完全相同的卡片。
  - 仅在当前已返回的关联数据范围内切换，不扩大内容接口。

## 需要先确认接口或产品规则

- [ ] 收藏更新提醒
  - 当收藏影片新增集数或状态更新时显示角标。
  - 需要先确认“更新”的权威字段、比较周期，以及是否需要站内通知。

- [ ] 稍后观看清单
  - 可先实现本地版；若要跨设备同步，需要新增账号侧的数据模型和接口。

- [ ] 分享到指定集数或播放进度
  - 需确认是否允许携带集数、是否必须从详情页重新做权限校验，以及播放器对深链接的支持范围。

## 建议实现顺序

1. 搜索历史与空结果回退
2. 本地观看记录单条删除
3. 详情页复制链接与剧情展开
4. 首页继续观看进度条与单条隐藏
5. 收藏页筛选与反馈快捷分类

每项完成后至少验证桌面和 390px 宽度；涉及账号、播放或写操作时，继续沿用现有同源会话、CSRF 与服务端权限校验。
