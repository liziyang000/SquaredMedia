# 豆瓣评分后台标签页集成设计

## 目标

让现有自定义菜单项 `豆瓣评分,admin/douban/index` 能够通过 `lbk-admin.php`
打开标准的 MacCMS 后台标签页。管理员无需离开后台框架，即可完成豆瓣接口配置、
评分校准、任务生成、Worker 执行、匹配结果核查和影片同步。

## 架构

为豆瓣插件增加后台模块控制器桥接。打包源文件位于
`addons/douban/bridge/DoubanAdmin.php`，声明
`app\admin\controller\Douban`，并将请求委托给现有插件控制器。
桥接控制器在委托前设置插件名、控制器和操作方法等路由信息，使现有插件视图和操作
接口可以继续复用。

部署时将桥接控制器安装为
`application/admin/controller/Douban.php`。现有前台模块桥接和
`/index.php/douban/index.html` 路由继续保留，用于兼容已有访问方式；正式后台管理入口
调整为 `/lbk-admin.php/admin/douban/index.html`。

## 后台行为

MacCMS 会把自定义菜单目标 `admin/douban/index` 转换为当前后台入口地址，并在右侧标签页
iframe 中打开。因为请求通过 `lbk-admin.php` 进入，插件视图中所有
`url('douban/...')` 生成的配置、同步和任务操作地址也会自动使用后台入口。

本方案不增加嵌套 iframe，也不修改自定义菜单控制器或模板。豆瓣管理页作为标准后台
标签页内容显示，不重复渲染左侧导航和后台顶栏，并适配标签页可用宽度。

管理员身份继续通过 MacCMS 原生 `Admin->checkLogin()` 校验。没有有效后台登录态的
直接请求仍返回“请先登录管理员账号”，不能执行配置或数据操作。

## 打包与部署

豆瓣插件发布包必须包含新的后台桥接控制器。部署脚本执行以下操作：

1. 如果 `application/admin/controller/Douban.php` 已存在，先生成备份。
2. 将发布包中的后台桥接控制器复制到该位置。
3. 保持现有前台桥接和公共豆瓣网关的安装逻辑不变。
4. 通过现有部署流程清理 MacCMS 缓存。

整个过程必须可重复执行，且不能改写管理员现有的自定义菜单配置。

## 验证

自动化测试检查后台命名空间、控制器继承关系、发布包文件和部署目标。生产环境验收会
创建一个短生命周期的 MacCMS 管理员会话，请求
`https://ping2.my/lbk-admin.php/admin/douban/index.html`，并确认：

- HTTP 状态码为 200，返回完整的豆瓣管理界面。
- 已登录请求不会出现“请先登录管理员账号”。
- 配置表单和 Worker 操作地址均使用 `lbk-admin.php/admin/douban/...`。
- 未登录请求仍然被拦截。

验收结束后立即销毁临时管理员会话。

## 非目标

- 不把豆瓣表单嵌入“自定义菜单配置”页面。
- 不修改 MacCMS 核心后台控制器、菜单解析逻辑或后台模板。
- 不删除现有前台模块兼容路由。
