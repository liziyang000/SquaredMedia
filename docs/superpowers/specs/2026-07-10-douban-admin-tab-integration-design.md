# Douban Admin Tab Integration Design

## Goal

Make the existing custom menu entry `豆瓣评分,douban/index` open the Douban
management interface as a normal MacCMS admin tab under `lbk-admin.php`.
Administrators must be able to configure the gateway, calibrate scores, create
tasks, run the worker, review matches, and synchronize videos without leaving
the admin shell.

## Architecture

Add an admin-module controller bridge to the Douban addon. The packaged source
will live at `addons/douban/bridge/DoubanAdmin.php`, declare
`app\admin\controller\Douban`, and delegate to the existing addon controller.
The bridge will set the addon/controller/action route values before delegation
so the current addon view and action methods continue to work.

Deployment will install the bridge as
`application/admin/controller/Douban.php`. The existing index-module bridge and
`/index.php/douban/index.html` route remain available for compatibility, but
the documented management entry becomes
`/lbk-admin.php/douban/index.html`.

## Admin Behavior

MacCMS already converts the custom menu target `douban/index` into the current
admin entry URL and opens it in the right-side tab iframe. Because the request
runs through `lbk-admin.php`, every `url('douban/...')` generated by the addon
view will also stay under the admin entry. No nested iframe and no changes to
the core quick-menu controller or template are required.

Authentication continues to use MacCMS `Admin->checkLogin()`. Direct requests
without an active admin session return the existing login-required response.
The standalone addon view remains responsible for its own content layout and
must fit the available tab width without adding another navigation shell.

## Packaging And Deployment

The Douban archive must include the new admin bridge. The deploy script will:

1. Back up an existing `application/admin/controller/Douban.php` file.
2. Copy the packaged admin bridge into that location.
3. Keep the existing index bridge and public gateway installation unchanged.
4. Clear MacCMS caches through the existing deployment flow.

The operation must be idempotent and must not rewrite the administrator's
custom-menu configuration.

## Verification

Automated verification will assert the admin namespace, controller inheritance,
archive entry, and deployment target. Production verification will create a
short-lived MacCMS admin session, request
`https://ping2.my/lbk-admin.php/douban/index.html`, and confirm:

- HTTP 200 with the complete Douban HTML interface.
- No login-required response for the authenticated request.
- Generated form and worker action URLs use `lbk-admin.php/douban/...`.
- An unauthenticated request remains blocked.

The temporary verification session will be destroyed immediately afterward.

## Non-Goals

- Do not embed the Douban form into the custom-menu configuration page.
- Do not modify MacCMS core admin controllers, menu parsing, or admin templates.
- Do not remove the existing index-module compatibility route.
